## AdsPower Control Strategy Decision

### Decision Made
**Primary Control Method:** AdsPower Local API (HTTP REST API)
**Endpoint:** `http://127.0.0.1:50325/api/v1`
**Reasoning:** Fastest, lightest, most efficient method for managing browser profiles

### Hardware Context
- Mac mini M2 Pro (10 cores, 32 GB RAM)
- AdsPower Global installed and running
- Capacity: 50-75 profiles realistically, 30-50 recommended

### Implementation Strategy

**Three-Tier Approach:**

1. **Primary (95% of use):** AdsPower API
   - Profile management (open, close, list, create, delete, update)
   - Proxy configuration
   - Fingerprint management
   - Batch operations

2. **Secondary (4% of use):** Clawdbot Browser Tool
   - Screenshots
   - Page interaction (forms, buttons)
   - Data extraction
   - JavaScript execution

3. **Tertiary (1% of use):** Puppeteer
   - Complex multi-step workflows
   - Advanced scraping
   - Custom automation

### Performance Targets

- Profile operations: <1 second via API
- Concurrent profiles: 25-50 (optimal), max 75-100
- Batch size: 15-20 profiles
- API rate: 100+ calls per minute
- Browser automation: 10-15 concurrent

### Best Practices

**For Speed:**
- Always use API for profile management
- Batch API calls when possible
- Parallel requests (10-20 concurrent)
- Cache profile lists

**For Resources:**
- Close profiles immediately after use
- Keep ≤25 profiles open simultaneously
- Monitor memory usage
- Use lightweight fingerprint configs

**For Stability:**
- Add error handling + retry logic (3 attempts)
- Exponential backoff for failures
- Monitor API response times
- Implement fallbacks

### Standard Workflow Pattern

```
1. GET /user/list → Get all profiles
2. Filter/select → Choose target profiles
3. POST /user/open → Open 15-20 profiles (parallel)
4. Browser tool → Automate tasks (parallel 10-15)
5. POST /user/close → Close 15-20 profiles (parallel)
```

### Key Commands

```bash
# List profiles
curl http://127.0.0.1:50325/api/v1/user/list

# Open profile
curl -X POST http://127.0.0.1:50325/api/v1/user/open \
  -H "Content-Type: application/json" \
  -d '{"user_id": "profile_id"}'

# Close profile
curl -X POST http://127.0.0.1:50325/api/v1/user/close \
  -H "Content-Type: application/json" \
  -d '{"user_id": "profile_id"}'
```

## Key Learnings Today

### Web Design
1. **Always use vector logos** (SVG) for external platforms - never emojis
2. **Official brand guidelines** are the source for authentic logos
3. **Professional credibility** depends on authentic branding

### Browser Automation
1. **API-first approach** is fastest and most efficient
2. **Tiered fallback strategy** provides flexibility
3. **Resource monitoring** critical for large-scale operations
4. **Batch operations** maximize throughput

### Project Management
1. **Document decisions** for future reference
2. **Save to memory** for cross-session persistence
